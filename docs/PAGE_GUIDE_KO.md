# 화면별 상세 사용 가이드

이 문서는 현재 저장소에 포함된 정적 페이지들의 역할과 실제 사용 방법을 화면 단위로 정리한 문서입니다.  
설명 기준은 현재 코드(`app/static/*.html`, `app/static/*.js`, `app/main.py`)입니다.

## 1. 공통 이해 사항

### 1.1 인증과 접근 제어

- 대부분의 화면은 로그인된 세션이 필요합니다.
- 인증이 필요한 API 호출에서 `401 Unauthorized`가 발생하면 클라이언트는 로그인 페이지로 이동합니다.
- 관리자만 접근해야 하는 화면은 `/static/admin.html`입니다.

### 1.2 사용자 역할

프로젝트 단위 역할은 크게 아래처럼 이해하면 됩니다.

- 관리자(`admin`)
  - 전체 프로젝트와 사용자 관리 가능
  - 대부분의 작업 수정 기능을 사용할 수 있음
- 프로젝트 소유자(`owner`)
  - 해당 프로젝트의 설정, 참가자, 작업 편집 가능
- 참가자(`editor` 또는 화면상 "참가자")
  - 해당 프로젝트의 작업 편집 가능
- 조회 전용(`viewer`)
  - 화면 조회 중심
  - 프로젝트 설정의 작업 편집 권한은 제한될 수 있음

실제 수정 가능 여부는 서버가 내려주는 `can_edit_tasks`, `can_manage_*` 성격의 권한 값과 현재 로그인 사용자에 따라 결정됩니다.

### 1.3 공통 작업 상세 모달

대시보드, 작업 보드, 간트 차트, 캘린더 등 여러 화면에서는 작업을 클릭하면 공통 작업 상세 모달이 열립니다.

이 모달에서 볼 수 있는 정보:

- 프로젝트명
- 대항목명
- 시작일
- 목표일
- 설명 본문

권한이 있으면 인라인 수정도 가능합니다.

수정 가능한 항목:

- 작업명
- 설명
- 대항목
- 시작일
- 목표일
- 상태(`Backlog`, `Upcoming`, `In Progress`, `Done`)

설명 본문 안의 URL은 자동으로 링크 처리됩니다.

### 1.4 공통 URL 파라미터

여러 프로젝트 화면은 `project_id`를 필수로 사용합니다.

자주 보이는 파라미터:

- `project_id`
  - 어떤 프로젝트를 열지 지정
- `checklist_id`
  - 특정 작업을 강조하거나 복귀 포커스를 맞출 때 사용
- `return_to`
  - 작업 관리 후 원래 화면으로 복귀시키기 위한 주소
- `task_action`
  - `create` 또는 `edit`
- `edit_checklist_id`
  - 설정 화면에서 바로 편집할 작업 ID
- `sort_by`, `task_sort`
  - 정렬 방식 유지
- `visible_tasks`
  - 필터에서 선택한 작업 ID 상태 유지
- `timeline_start`, `timeline_end`
  - 간트 범위 고정
- `month`
  - 캘린더 기준 월 지정

## 2. 화면 목록 요약

| 화면 | URL | 주 용도 |
| --- | --- | --- |
| 로그인 | `/static/login.html` | 로그인, 회원가입 |
| 메인 대시보드 | `/` | 프로젝트 요약, 빠른 이동, 프로젝트 생성 |
| 작업 보드 | `/static/project.html?project_id=...` | 칸반 보드 중심 작업 진행 |
| 프로젝트 설정 | `/static/project_settings.html?project_id=...` | 프로젝트 상세 관리, 작업 추가/관리 |
| 프로젝트 간트 | `/static/project_gantt.html?project_id=...` | 일정 시각화, 범위/필터/상세 수정 |
| 프로젝트 캘린더 | `/static/project_calendar.html?project_id=...` | 월간 일정 확인, 상세 수정 |
| 템플릿 관리 | `/static/template.html` | 템플릿 생성/편집/복원/삭제 |
| 선택 템플릿 백업 | `/static/template_export.html` | 일부 템플릿만 골라 JSON 백업 |
| 내 설정 | `/static/settings.html` | 내 정보/비밀번호/테마 색상 |
| 관리자 페이지 | `/static/admin.html` | 사용자/프로젝트/브랜딩/가입 코드 관리 |

## 3. 로그인 화면

### 3.1 URL

- `/static/login.html`

### 3.2 목적

- 기존 사용자 로그인
- 가입 코드 기반 신규 사용자 등록

### 3.3 주요 구성

- 로그인 폼
  - 아이디
  - 비밀번호
- 회원가입 폼
  - 가입 아이디
  - 이름
  - 가입 코드
  - 비밀번호
  - 비밀번호 확인

### 3.4 동작 방식

- 로그인 성공 시:
  - 관리자면 `/static/admin.html`로 이동
  - 일반 사용자면 `/`로 이동
- 회원가입 성공 시:
  - 바로 같은 계정으로 자동 로그인
  - 이후 관리자 여부에 따라 적절한 화면으로 이동

### 3.5 유의 사항

- 회원가입은 서버에 저장된 가입 코드와 일치해야 합니다.
- 비밀번호 확인이 다르면 회원가입 요청 전 프론트에서 차단됩니다.
- 초기 관리자 계정은 서버 환경 변수 기준으로 생성됩니다.

## 4. 메인 대시보드

### 4.1 URL

- `/`

### 4.2 목적

- 전체 프로젝트 현황 확인
- 내 마감 임박 작업 확인
- 프로젝트 생성
- 프로젝트별 상세 화면으로 이동

### 4.3 상단 영역

- 사용자 정보 표시
- 템플릿 페이지 이동
- 내 설정 이동
- 관리자면 `Admin` 링크 표시
- 로그아웃 버튼

대시보드 제목과 부제는 관리자 페이지에서 수정할 수 있습니다.

### 4.4 통계 카드

표시 항목:

- 전체 프로젝트 수
- 진행 중 프로젝트 수
- 전체 작업 수
- 완료율

이 값은 `/api/dashboard` 결과를 사용합니다.

### 4.5 내 프로젝트 마감 임박 작업

필터:

- 전체
- Owner
- 참가자

각 작업 카드에서 할 수 있는 일:

- 작업명을 눌러 상세 모달 열기
- 권한이 있으면 모달 안에서 인라인 수정
- `프로젝트 보기` 버튼으로 해당 프로젝트 작업 보드로 이동
  - 이때 `checklist_id`가 함께 전달되어 해당 작업을 강조할 수 있음

### 4.6 프로젝트 일정 간트 차트

대시보드에는 프로젝트 단위의 요약 간트 차트가 있습니다.

가능한 동작:

- 범위 직접 입력
- 자동 범위 재설정
- 프로젝트별 표시/숨김
- 정렬 변경
- 프로젝트 바 클릭 시 프로젝트 간트 차트 페이지로 이동

이 영역은 프로젝트 일정의 개요를 보는 용도이며, 세부 작업 관리의 중심은 프로젝트 간트 화면과 프로젝트 설정 화면입니다.

### 4.7 프로젝트 등록

입력 항목:

- 프로젝트명
- Owner username
- 마감일
- 상태
- 설명

생성 후:

- 목록과 통계가 즉시 갱신됩니다.
- 로컬에서는 현재 로그인 사용자 아이디가 Owner 필드 기본값으로 채워집니다.

### 4.8 프로젝트 목록

각 프로젝트 카드에서 가능한 이동:

- `작업 보드`
- `간트 차트`
- `캘린더`
- `프로젝트 설정`

삭제 가능 조건:

- 관리자이거나 프로젝트 owner

삭제 시 추가 확인:

- 삭제 확인 다이얼로그
- 본인 비밀번호 입력

프로젝트를 삭제하면 관련 작업도 함께 삭제됩니다.

## 5. 프로젝트 작업 보드

### 5.1 URL

- `/static/project.html?project_id={id}`

선택 파라미터:

- `checklist_id`
  - 특정 작업 카드 강조

### 5.2 목적

- 프로젝트 작업을 칸반 보드 형태로 진행 상태별 관리

### 5.3 화면 구성

- 프로젝트 제목
- 간트 차트 이동
- 캘린더 이동
- `작업 추가/관리 열기` 버튼
- 칸반 컬럼 4개
  - `Backlog`
  - `Upcoming`
  - `In Progress`
  - `Done`

### 5.4 주요 사용 흐름

1. 작업 카드 확인
2. 카드 클릭으로 상세 모달 열기
3. 권한이 있으면 모달 안에서 작업 내용 수정
4. 권한이 있으면 드래그앤드롭으로 컬럼 이동

### 5.5 드래그앤드롭 규칙

- 권한이 있는 사용자만 드래그 가능
- 컬럼으로 드롭하면 `workflow_status`가 변경됩니다
- `Done`으로 이동하면 `is_done`도 함께 완료 상태로 반영됩니다

### 5.6 작업 상세 모달

보드에서 열리는 모달은 인라인 수정형입니다.

수정 가능한 대표 항목:

- 작업명
- 설명
- 대항목
- 시작일
- 목표일
- 상태

### 5.7 작업 추가/관리 버튼

- 프로젝트 설정 화면의 `#task-management` 영역으로 이동합니다
- 대량 수정, 대항목 관리, 참가자 관리까지 한 번에 하려면 이 버튼을 쓰는 편이 더 적합합니다

## 6. 프로젝트 설정 화면

### 6.1 URL

- `/static/project_settings.html?project_id={id}`

자주 쓰는 파라미터:

- `return_to`
- `task_action=create|edit`
- `edit_checklist_id`
- `task_sort`

### 6.2 목적

- 프로젝트 자체 정보와 권한, 참가자, 작업 구조를 가장 자세하게 관리하는 중심 화면

### 6.3 기본 정보 섹션

수정 가능 항목:

- 프로젝트명
- Owner
- 마감일
- 상태
- 설명

저장 시:

- 대시보드, 보드, 간트, 캘린더 등 관련 화면에 공통 반영

### 6.4 프로젝트 참가자 섹션

가능한 작업:

- 참가자 추가
- 역할 지정
  - `editor`
  - `viewer`
- 기존 참가자 역할 변경
- 참가자 제거

규칙:

- Owner는 항상 참가자로 포함됩니다
- Viewer는 주로 조회 전용입니다

### 6.5 템플릿 적용 섹션

기능:

- 기존 템플릿을 선택해 현재 프로젝트에 적용

주의:

- 적용 시 현재 작업 구조를 덮어쓸 수 있으므로 확인 절차가 있습니다
- 적용 결과는 작업 관리, 보드, 간트, 캘린더에 반영됩니다

### 6.6 작업 추가/관리 섹션

이 화면의 핵심 영역입니다.

구성:

- 좌측: 대항목 관리
- 우측: 작업 항목 관리

#### 대항목 관리

가능한 작업:

- 대항목 추가
- 대항목 이름 변경
- 대항목 삭제
- 드래그로 대항목 순서 변경

#### 작업 항목 관리

가능한 작업:

- 새 작업 추가
- 기존 작업 수정
- 작업 삭제
- 완료 체크
- 시작일/목표일/상태 수정
- 설명 내용 수정
- 드래그로 순서 변경

정렬 옵션:

- 기존 정렬
- 이름순
- 시작일 빠른순
- 종료일 빠른순

`기존 정렬`에서는 드래그 정렬이 가능하고, 다른 정렬 기준에서는 보기 기준 정렬로 전환됩니다.

### 6.7 복귀 배너(`return_to`)

간트나 캘린더에서 이 화면으로 들어온 경우 상단에 복귀 배너가 표시될 수 있습니다.

동작:

- 어떤 화면에서 왔는지 안내
- 작업 저장 후 원래 화면으로 자동 복귀 가능
- 특정 작업 편집 모드로 바로 진입 가능

## 7. 프로젝트 간트 차트

### 7.1 URL

- `/static/project_gantt.html?project_id={id}`

주요 파라미터:

- `timeline_start`
- `timeline_end`
- `visible_tasks`
- `sort_by`
- `checklist_id`

### 7.2 목적

- 작업의 시작일과 목표일을 시간축 위에서 시각적으로 확인

### 7.3 상단 이동/관리 요소

- 작업 보드 이동
- 캘린더 이동
- 상단 링크 `작업 추가/관리`
- 안내 배너 `작업 추가/관리 열기`

즉, 간트 화면 안에서도 바로 작업 관리 중심 화면으로 갈 수 있습니다.

### 7.4 간트 범위 제어

가능한 작업:

- 시작일/종료일 직접 입력
- 자동 범위 재설정

직접 입력 시:

- 두 날짜 모두 필요
- 시작일이 종료일보다 늦으면 차단

### 7.5 필터 패널

지원 기능:

- 카드형 / 표형 레이아웃 전환
- 작업 정렬
- 전체 선택
- 전체 해제
- 개별 작업 체크박스 선택

선택 상태는 URL 파라미터에 반영되어 새로고침 후에도 유지될 수 있습니다.

### 7.6 본문 영역

표시 내용:

- 작업별 대항목 색상
- 일정 막대(bar)
- 오늘 마커
- 프로젝트 마감 마커
- 레이아웃에 따라 카드형 또는 테이블형 표시

### 7.7 작업 상세 수정

가능한 방식:

- 간트의 작업 요소 클릭
- 범위 미설정 작업 카드 클릭

권한이 있으면 상세 모달 안에서 바로 수정 가능:

- 작업명
- 설명
- 대항목
- 시작일
- 목표일
- 상태

### 7.8 범위 미설정 작업

시작일 또는 목표일이 충분히 설정되지 않은 작업은 별도 목록으로 분리되어 표시됩니다.

이 목록에서도 상세 모달을 열 수 있습니다.

## 8. 프로젝트 캘린더

### 8.1 URL

- `/static/project_calendar.html?project_id={id}`

주요 파라미터:

- `month`
- `visible_tasks`
- `sort_by`
- `checklist_id`

### 8.2 목적

- 작업을 월간 캘린더 형태로 확인

### 8.3 상단 이동/관리 요소

- 작업 보드 이동
- 간트 차트 이동
- 상단 링크 `작업 추가/관리`
- 안내 배너 `작업 추가/관리 열기`

즉, 캘린더 화면에서도 바로 프로젝트 설정의 작업 관리 영역으로 연결됩니다.

### 8.4 월 이동 기능

지원 요소:

- 이전 달
- 기준 월 선택
- 이번 달
- 다음 달

선택한 월은 URL의 `month` 값에 반영됩니다.

### 8.5 필터 패널

지원 기능:

- 정렬
- 전체 선택
- 전체 해제
- 작업별 체크박스 선택

필터 결과는 캘린더 본문과 일정 미설정 작업 목록에 함께 반영됩니다.

### 8.6 캘린더 본문

각 날짜 셀에서 볼 수 있는 정보:

- 날짜
- 오늘 뱃지
- 프로젝트 마감 뱃지
- 해당 날짜에 걸친 작업 칩

작업 칩을 클릭하면 상세 모달이 열립니다.

### 8.7 일정 범위 계산 방식

캘린더는 작업의 시작일과 목표일을 기준으로 범위를 계산합니다.

- 시작일과 목표일이 모두 있으면 해당 구간에 표시
- 목표일만 있고 시작일이 없으면 오늘 날짜를 기준으로 자동 보정해 표시하는 로직이 일부 적용됩니다
- 범위를 만들 수 없는 작업은 아래의 미설정 목록으로 분리됩니다

### 8.8 일정 미설정 작업

별도 카드 목록으로 표시되며, 여기서도 상세 모달 열기와 수정이 가능합니다.

## 9. 템플릿 관리 화면

### 9.1 URL

- `/static/template.html`

### 9.2 목적

- 프로젝트에 재사용할 작업 템플릿 생성/편집
- 백업/복원

### 9.3 템플릿 생성

입력 항목:

- 템플릿 이름
- 템플릿 설명

생성 후 목록에서 선택해 상세 편집으로 들어갈 수 있습니다.

### 9.4 템플릿 백업/복원

백업 기능:

- 전체 템플릿 즉시 백업
- 선택 템플릿 백업 페이지로 이동

복원 기능:

- JSON 파일 업로드
- 동일 이름 템플릿 처리 방식 선택
  - `overwrite`
  - `skip`

### 9.5 템플릿 목록

지원 기능:

- 템플릿 단건 선택
- 전체 선택
- 선택 개수 표시
- 선택 템플릿 일괄 삭제

### 9.6 템플릿 편집 패널

템플릿을 선택하면 상세 편집 패널이 열립니다.

가능한 작업:

- 템플릿 이름/설명 수정
- 템플릿 삭제
- 대항목 추가/이름 변경/삭제/순서 조정
- 작업 항목 추가/수정/삭제

구조적으로 프로젝트 설정 화면의 작업 관리와 유사하지만, 실제 프로젝트가 아니라 템플릿 원본을 수정하는 점이 다릅니다.

### 9.7 변경사항 관리

- 저장되지 않은 변경사항이 있는 상태에서 다른 템플릿으로 이동하려 하면 확인 절차가 표시됩니다
- 저장 시 템플릿 메타 정보와 대항목/작업 항목 전체 구조가 함께 갱신됩니다

## 10. 선택 템플릿 백업 화면

### 10.1 URL

- `/static/template_export.html`

### 10.2 목적

- 전체 백업이 아니라 일부 템플릿만 골라 JSON으로 내보내기

### 10.3 주요 기능

- 템플릿 목록 조회
- 전체 선택
- 개별 선택
- 선택 개수 표시
- 새로고침
- 선택 템플릿 백업

### 10.4 백업 동작

- 선택된 템플릿들의 대항목과 작업 항목을 묶어 JSON 파일로 다운로드합니다
- 서버 전용 export API가 없거나 실패하면 클라이언트에서 직접 스테이지/아이템을 모아 fallback payload를 구성합니다

## 11. 내 설정 화면

### 11.1 URL

- `/static/settings.html`

### 11.2 목적

- 내 프로필 수정
- 비밀번호 변경
- 개인 테마 색상 저장

### 11.3 페이지 테마 색상

지원 기능:

- 새 비밀번호
- 새 비밀번호 확인

새 비밀번호 확인 불일치 시 프론트에서 먼저 차단합니다.
